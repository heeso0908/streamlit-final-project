# 피드백 분류 Skill

주어진 피드백 CSV를 읽고 Claude가 직접 각 행을 분류해 유형·감정·부유형 열을 추가한 새 CSV로 저장한다.

## 인수
`$ARGUMENTS` — 분류할 CSV 파일 경로 (생략 시 `Day3_과제_feedback.csv` 사용)

## 분류 우선순위 규칙

**유형 (주유형)**
1. 불만: 별점 1~2점 OR 부정 감정 표현 포함 (다른 유형보다 우선)
2. 요청: "~하면 좋겠다 / 확인해달라 / 해주세요 / 있으면 좋겠다" 등 개선·추가 요청 (불만 아닐 때)
3. 칭찬: 긍정적 감상·감사 표현
4. 문의: 사실 확인을 위한 질문형

**부유형**: 불만이면서 동시에 요청 표현이 있으면 "요청", 없으면 빈칸

**감정**
- 긍정: 만족·칭찬·감사
- 부정: 불만·실망·불편 (별점 1~2면 무조건 부정)
- 중립: 단순 질문·중립적 내용
- 별점 있으면 별점 우선, 없으면 내용으로 추정

## Few-shot 기준 예시
- 별점=1, "20분 기다렸어요. 너무 오래 걸려요." → 불만 / 부정 / 부유형 없음
- 별점=2, "너무 달아요. 당도 조절 옵션 있으면 좋겠어요." → 불만 / 부정 / 부유형=요청
- 별점=없음, "비건 디저트도 있나요?" → 문의 / 중립 / 부유형 없음
- 별점=5, "아메리카노 맛있어요. 단골 됐어요!" → 칭찬 / 긍정 / 부유형 없음
- 별점=없음, "앱에서 오류나요. 확인 부탁드려요." → 불만 / 부정 / 부유형=요청

## 실행 절차

1. `$ARGUMENTS`로 지정된 CSV 파일을 Read 도구로 읽는다. 인수가 없으면 `Day3_과제_feedback.csv`를 읽는다.
2. CSV의 모든 행을 위 규칙과 예시를 기준으로 Claude가 직접 판단해 분류한다.
   - 경계 케이스는 별점과 내용의 주된 의도를 함께 고려해 단일 유형으로 결정한다.
3. 분류 결과를 표로 보여주고, 기존 분류와 다른 행이 있으면 "변경" 표시를 붙인다.
4. 원본 파일명에서 확장자 앞에 `_분류`를 붙인 파일명(예: `feedback_분류.csv`)으로 결과를 저장한다.
   - 열 순서: id, 받은날짜, 경로, 별점, 내용, 유형, 감정, 부유형
5. 저장 완료 후 변경 건수와 유형별 집계를 한 줄로 요약한다.
